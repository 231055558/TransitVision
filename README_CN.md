# TransitVision: 高精度公交客流OD识别系统

TransitVision 是一套专业级的公共交通客流分析解决方案。提供端到端的乘客**起讫点(OD)识别**、实时**拥挤度分析**和高精度**上下客计数**，专门针对公交车载监控的极端场景设计。

## 🚀 核心创新与亮点

我们的系统专门为克服公交车载监控的独特挑战而设计：

- **公交场景专用数据增强**：针对性的图像预处理和数据增强管道，能够处理**极端光照变化**、**色彩失真**和**严重画面抖动**等公交运行中常见的问题。
- **精细调参的YOLO11x-Seg与ReID**：深度优化的YOLO11x分割模型用于精确的人员/门框掩码提取，以及定制化的ReID超参数调整，确保在低质量成像条件下的鲁棒性。
- **Pose2ID方案集成**：引入**Pose2ID**机制，利用姿态感知的特征提取，显著提升不同相机视角间的ReID准确度（如前门上车视角与后门下车视角的匹配），有效消除"相机视角偏差"。
- **工业级部署**：已成功应用于城市公交系统，用于公交网络优化和站点吞吐量分析。

## 🧠 核心算法逻辑

TransitVision 采用三个精心设计的逻辑判断算法，确保在复杂场景下的高精度识别：

### 1. 自适应门框预处理 (`door_preprocessor.py`)
相比静态ROI方案，我们的系统动态分割公交车门：
- **去噪与连通域过滤**：使用连通域(CC)分析从背景噪声中隔离门框结构。
- **几何校正**：自动计算门框旋转角度，应用仿射变换对齐坐标系，确保无论相机安装倾角如何都能保持一致的空间逻辑。

### 2. 模式识别的上客检测 (`boarding_counter.py`)
区分真实乘客与车外行人的关键算法：
- **空间转移模式分析**：追踪人物bbox与门框掩码的"重叠比例"，识别特定模式（车外→车内→穿过门中心）。
- **高度约束过滤**：实现物理约束检查。车内乘客的平均高度必须达到门框高度的48%以上，有效过滤来自街道水平面的误检。

### 3. V3版本下客逻辑与位运算优化 (`alighting_counter.py`)
针对后门高密度下客场景的精细设计：
- **矩阵位运算检测**：不使用简单的bbox检查，而是采用**人物多边形 + 门框掩码**的位运算AND操作。这种$O(1)$复杂度的方案实现像素级精度的进门检测。
- **消失宽限期机制**：处理追踪碎片化的关键逻辑。若乘客在显示"下车意图"后于门框区域消失，在宽限期内仍被计数，提供对遮挡的鲁棒性。

### 4. 端到端OD匹配 (`main.py`)
- **乘客生命周期数据库**：维护"在车上"乘客的完整生命周期。
- **双阶段匹配策略**：
    1. **即时匹配**：下车时使用ReID特征进行实时匹配。
    2. **最终交叉匹配**：行程结束后执行全局贪心匹配优化，闭环"待定"乘客，最大化**闭环率**。

## 📊 系统输出

系统为每次运行生成丰富的分析数据：
- **匹配乘客JSON**：每个识别个体的上下车站点关联。
- **拥挤度曲线**：整条路线的车内满载度可视化。
- **站点统计**：每站的详细上下客人数。
- **视觉证据**：每个上下客事件的捕获图像，用于审计。
